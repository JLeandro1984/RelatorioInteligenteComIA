1. Visão Geral do Projeto

Objetivo

Criar uma plataforma de business intelligence com interface universal, motor de IA simulada e geração automática de insights, utilizando apenas HTML, CSS e JavaScript puro.

Stack Tecnológico

JavaScript ES6+ — Lógica, modularização, eventos

Padrão Arquitetural

  • Singleton Modules: Cada motor é um IIFE (Immediately Invoked Function Expression)
  • Configuration over Convention: Relatórios dirigidos por config declarativa
  • Separação de Responsabilidades: Config, Dados, Lógica e Apresentação desacoplados
  • Lazy Evaluation: Insights gerados sob demanda
  • Backward Compatible: Pronto para trocar JSON por API REST sem quebrar código

3. Métodos Mais Importantes

Top 5 Métodos Críticos

# Método Arquivo Responsabilidade
1 generateReport() app.js Orquestra TODO o fluxo: filtros → IA → render
2 gerarInsights() insight-engine.js Motor de IA — gera insights contextuais
3 enriquecerDados() report-engine.js Pipeline de dados brutos → filtrados
4 render() chart-engine.js Entry point para todos os gráficos
5 REPORT_CONFIGS report-config.js Configuração universal de relatórios

app.js — Orquestrador

selectReport(key): Switcheia entre relatórios. Carrega filtros e sugestões dinâmicas.

generateReport(): Fluxo principal — coleta filtros, enriquece dados, gera insights via IA, renderiza tudo.

loadData(): Carrega JSON com dados brutos (fetch assíncrono).

init(): Bootstrap da aplicação — carrega dados, monta sidebar, liga listeners.

insight-engine.js — IA

gerarInsights(reportKey, dados, filtros, pergunta): Núcleo da IA — seleciona gerador por tipo, interpreta pergunta, retorna insights prioritários.

interpretarPergunta(pergunta, contexto): NLP simples — mapeia pergunta a intenção via palavras-chave.

calcularKPIs(reportKey, dados): Computar valores dos KPI cards.

gerarResumo() / gerarRecomendacoes(): Templates profissionais por tipo de relatório.

4. Motor de IA Simulada

Camadas da IA

  1. Interpretação de Intenção: Busca palavras-chave na pergunta do usuário
  2. Análise de Dados: Processa dados brutos via analisadores (vendas, estoque, etc)
  3. Geração de Insights: Transforms métricas em narrativa profissional
  4. KPIs Calculados: Números dos cards de resumo
  5. Resumo Executivo: Parágrafo profissional de contexto
  6. Recomendações: Ações sugeridas baseadas em regras

Intenções Mapeadas

Intenção Palavras-chave Contextos
mais_vendido mais vendido, produto top, melhor produto produtos, vendas
estoque_baixo estoque baixo, falta, acabou, critico estoque, produtos
cliente_top cliente mais comprou, maior cliente clientes, vendas
maior_nf maior nota, nota fiscal maior, maior valor notasFiscais, vendas
funcionario_destaque funcionário, vendedor, mais pedidos funcionarios, vendas
margem margem, lucro, rentabilidade produtos
✓ Vantagem: IA 100% offline, sem API externa. Pronta para integração futura com GPT-4, Claude ou Gemini via backend.

6. Como Rodar o Projeto

Opção 1: Live Server (Recomendado)

  1. Instale extensão "Live Server" no VS Code
  2. Clique direito em index.html → "Open with Live Server"
  3. Abre automaticamente em http://localhost:5500

Opção 2: Python (Qualquer máquina)

cd "caminho\do\projeto"
python -m http.server 8000
# Acesse: http://localhost:8000

Opção 3: .bat (Windows)

Duplo-clique em START_SERVER.bat — inicia servidor Python automaticamente.

⚠️ Importante: Use servidor local (http://...). Abrir direto o arquivo (file://...) bloqueará o fetch do JSON por CORS.

7. Como Adicionar um Novo Relatório

Passo 1: Dados (report-data.json)

Adicione a entidade raiz se não existir:

{
  "minhaEntidade": [
    { "id": "001", "nome": "Item", "status": "ativo" }
  ]
}

Passo 2: Configuração (report-config.js)

Insira objeto em REPORT_CONFIGS:

{
  key: 'minhaEntidade',
  label: 'Minha Entidade',
  icon: 'folder',
  color: 'blue',
  filtros: [ { campo: 'status', ... } ],
  kpis: [ { key: 'total', label: 'Total', ... } ],
  colunas: [ { campo: 'id', tipo: 'text' } ],
  graficos: ['bar', 'pie'],
  perguntasSugeridas: ['qual item tem mais atividade?']
}

Passo 3: (Opcional) IA Customizada

Se precisar lógica específica em insight-engine.js:

// Adicione handler em intentHandlers:
getMeuInsight(dados) {
  return {
    tipo: 'info', icone: 'lightbulb',
    titulo: 'Meu Insight',
    texto: 'Análise customizada',
    valor: '123 itens'
  };
}

// Registre intenção em INTENCOES:
{
  id: 'meu_insight',
  palavrasChave: ['minha pergunta'],
  contextos: ['minhaEntidade'],
  handler: 'getMeuInsight'
}

Pronto! 🎉

Nenhum outro arquivo precisa ser alterado. Sidebar mostrará novo relatório automaticamente.

8. Fluxo de Dados (Happy Path)

1. init()
   ├─ loadData() → JSON carregado
   ├─ buildSidebarMenu() → Sidebar renderizada
   └─ bindEvents() → Listeners ativos

2. Usuário clica "Clientes"
   └─ selectReport('clientes')
      ├─ Carrega config do relatório
      ├─ Monta filtros dinâmicos
      └─ Monta sugestões de perguntas

3. Usuário clica "Gerar Relatório"
   └─ generateReport()
      ├─ collectFilters() → Filtros aplicados
      ├─ ReportEngine.enriquecerDados() → Dados filtrados
      ├─ InsightEngine.gerarInsights() → Insights IA
      ├─ InsightEngine.calcularKPIs() → Valores KPI
      ├─ InsightEngine.getChartData() → Dados gráficos
      ├─ ReportEngine.renderKPIs() → Cards
      ├─ ReportEngine.renderInsights() → Insights
      ├─ ReportEngine.renderCharts()
      │  └─ ChartEngine.render() → new Chart()
      ├─ ReportEngine.renderTable() → Tabela interativa
      ├─ ReportEngine.renderSummary() → Resumo
      └─ ReportEngine.renderRecommendations() → Recomendações

4. Usuário pesquisa na tabela
   └─ ReportEngine.filterTable(query)
      └─ Tabela atualizada em tempo real

12. Performance e Otimizações

  • Zero dependências: CSS/JS nativos, sem build tools
  • Gráficos lazy: Destroi instância anterior antes de recriar
  • Animações GPU: Usar transform + opacity para máximo performance
  • Lazy loading de ícones: Lucide carregado via CDN
  • Debouncing na tabela: Busca otimizada
  • Event delegation: Menos listeners, melhor memória
  • Single render cycle: Insights gerados uma única vez por requisição
